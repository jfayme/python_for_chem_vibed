**Two hard constraints (apply to every decision below):**

## A2. matplotlib house style — `STYLE SETUP` cell

Add this as the **first code cell** of **Notebook 06 and Notebook 07 only** (they are the only notebooks that plot). Label the markdown cell above it `## Setting up our plotting style` with one sentence: *"Run this once at the top — it gives every figure in this notebook a clean, colour-blind-friendly house style."*

```python
import matplotlib.pyplot as plt
from cycler import cycler

# Python-for-Chemists house style — colour-blind safe (Okabe–Ito)
OKABE_ITO = ["#0072B2", "#D55E00", "#009E73", "#E69F00", "#CC79A7", "#56B4E9"]

plt.rcParams.update({
    "figure.figsize":    (7.2, 4.5),
    "figure.dpi":        120,
    "savefig.dpi":       200,
    "savefig.bbox":      "tight",
    "font.family":       "DejaVu Sans",      # ships with matplotlib — nothing to install
    "font.size":         13,
    "axes.titlesize":    16,
    "axes.titleweight":  "bold",
    "axes.labelsize":    13,
    "axes.prop_cycle":   cycler(color=OKABE_ITO),
    "axes.spines.top":   False,
    "axes.spines.right": False,
    "axes.grid":         True,
    "grid.color":        "#E4E0D6",
    "grid.linewidth":    0.8,
    "axes.edgecolor":    "#5E6B66",
    "axes.linewidth":    1.1,
    "xtick.color":       "#1C2B27",
    "ytick.color":       "#1C2B27",
    "xtick.labelsize":   12,
    "ytick.labelsize":   12,
    "legend.frameon":    False,
})

# Named colours for explicit use (e.g. drug-likeness)
PASS, FAIL = "#0072B2", "#D55E00"   # blue / vermillion — never red/green
```

**Notes**
- Keep `font.family="DejaVu Sans"` — it is matplotlib's built-in font, so there is nothing to install and the "everything just runs" promise holds. Do **not** switch to a font that needs installing.
- **Do not** put this cell in Notebooks 01–05. Those do not plot, and `import matplotlib` before imports are taught (L03) would be confusing magic.
- Every figure the course produces must have: a **title**, **unit-bearing axis labels** (e.g. `"Wavelength / nm"`, `"Molar mass / g·mol⁻¹"`), and a **legend** whenever there is more than one series.
- The optional `chem.mplstyle` file (shipped alongside this addendum) is the same settings as a reusable style sheet — see Appendix at the end. The inline block above is the **primary, recommended** approach because it has no file-path pitfalls for beginners.

## A4. Callouts — portable, colour-coded asides

Replace the current asides (which are distinguished only by a topical emoji) with a **blockquote + coloured-circle marker** system. The coloured circle is a Unicode emoji, so the colour-coding survives even on GitHub where CSS is stripped. Keep the **bold label line**; the circle's colour maps to the palette role.

Add a builder helper and route **every** aside through it:

```python
def callout(kind, title, body_lines):
    """kind in {'chem','rdkit','try','solution'} -> markdown blockquote."""
    marker = {"chem": "🟢", "rdkit": "🔵", "try": "🟠", "solution": "🟣"}[kind]
    head = f"> {marker} **{title}**"
    body = "\n".join(f"> {line}" for line in body_lines)
    return f"{head}\n>\n{body}"
```

The four types and their exact markdown:

```
> 🟢 **Chemistry aside — <topic>**
>
> <1–3 plain-language sentences>

> 🔵 **With RDKit**
>
> <the one-liner reveal that matches the hand-built version>

> 🟠 **Try it yourself**
>
> <the exercise prompt — the empty/partial code cell follows>

> 🟣 **Solution**
>
> <a sentence of worked explanation — the solution code cell follows>
```

**Mapping from the old markers:** `🧪 Chemistry aside` → `🟢 Chemistry aside`; `⚗️ With RDKit` → `🔵 With RDKit`; `🔬 Try it yourself` → `🟠 Try it yourself`; `Solution` → `🟣 Solution`. Drop the old `🧪 ⚗️ 🔬` glyphs — the coloured circles now carry the meaning, and one consistent system reads more cleanly than three competing emoji.

Colour → meaning, taught implicitly and consistently across all seven notebooks: **🟢 green = chemistry, 🔵 blue = RDKit, 🟠 orange = your turn, 🟣 purple = solution.**

## A5. Notebook header — wayfinding

Every notebook's **title cell** becomes a consistent, portable header. Keep the existing `# Lecture N — <Topic>` and one-line subtitle, and add a progress strip beneath it. Builder helper:

```python
def header_md(n, topic, subtitle):
    dots = "●" * n + "○" * (7 - n)     # e.g. ●●●○○○○
    return (f"# Lecture {n:02d} — {topic}\n\n"
            f"*{subtitle}*\n\n"
            f"**Lecture {n} of 7** · ≈ 60 min  \n"
            f"{dots}")
```

(The two trailing spaces after `≈ 60 min` are deliberate — they force the line break in markdown so the dots sit on their own line.) Renders identically everywhere as plain text + Unicode.

## Appendix — optional `chem.mplstyle` (the DRY upgrade)

A file named `chem.mplstyle` is shipped with this addendum. It is the A2 settings as a reusable matplotlib style sheet. **Optional**, for later: commit it at the repo root, then replace the inline `plt.rcParams.update({...})` in the setup cell with a robust loader that finds it regardless of where Jupyter was launched:

```python
import matplotlib.pyplot as plt
from pathlib import Path

# find chem.mplstyle in the current dir or any parent (repo root)
for _p in [Path.cwd(), *Path.cwd().parents]:
    _style = _p / "chem.mplstyle"
    if _style.exists():
        plt.style.use(str(_style))
        break
```

Stick with the inline block (A2) if you want the simplest, most foolproof thing for absolute beginners — both produce identical figures.
